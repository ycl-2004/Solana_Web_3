可以，取消就是先 **deactivate stake**。
你这次的 **Stake Account** 是：

```text
HWVdiFgSJHAxuVxRsv7HcR1ZSkn7LU8kDwDoLzKYx71B
```

而 Solana 的规则是：

* 先 **deactivate / undelegate**
* stake 会进入 **deactivating / cooling down**
* 等它变成 **inactive**
* 才能 **withdraw** 回你的钱包 ([Solana][1])

## 你现在直接这样做

先确认你在 Devnet：

```bash
solana config get
```

如果不是，就切到 Devnet：

```bash
solana config set --url https://api.devnet.solana.com
```

Devnet 的官方公开 RPC 就是这个。 ([Solana][2])

然后取消委托：

```bash
solana deactivate-stake HWVdiFgSJHAxuVxRsv7HcR1ZSkn7LU8kDwDoLzKYx71B
```

这个命令是官方支持的 stake CLI 操作之一。 ([docs.solana.com][3])

## 怎么看有没有取消成功

查 stake account 状态：

```bash
solana stake-account HWVdiFgSJHAxuVxRsv7HcR1ZSkn7LU8kDwDoLzKYx71B
```

如果它还在冷却中，你会看到类似 deactivating / cooling down 的状态；官方说明里也写到，刚 un-delegate 后不会立刻可提领。 ([Solana][4])

## 之后怎么拿回钱包

等它 inactive 后，再提回主钱包：

```bash
solana withdraw-stake HWVdiFgSJHAxuVxRsv7HcR1ZSkn7LU8kDwDoLzKYx71B /Users/yichenlin/.config/solana/id.json ALL
```

`withdraw-stake` 也是官方支持的 stake CLI 命令。只有未委托 / inactive 的 stake 才能被提领。 ([docs.solana.com][3])

## 你这笔为什么能取消

因为你刚刚输出里已经显示：

* `staker = StXNKB...`
* `withdrawer = StXNKB...`

也就是你的主钱包同时有 **stake authority** 和 **withdraw authority**，所以你有权取消和提领。stake account 本质上是你自己的账户，不是把钱送给 validator。 ([Solana][1])
